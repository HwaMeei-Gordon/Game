# THE TOWER · 無盡塔防 — 完整設計說明書

> 本文件給開發者與 AI 閱讀，描述遊戲的**所有機制、數值、存檔格式、程式架構與設計理念**，
> 並記錄**玩家（產品擁有者）對遊戲的期望與路線圖**。遊戲畫面本身不顯示這些內容。

---

## 1. 遊戲總覽

- 類型：單人、無盡波次（endless wave）塔防 roguelite。
- 視角：玩家就是場地正中央的「塔」。敵人從外圈四面八方生成，往中心逼近，碰到塔就對塔造成傷害。
- 兩層成長：
  - **局內（金幣）**：每場遊戲內擊殺/過波獲得金幣，即時購買升級（攻擊力、攻速、生命…），死亡後歸零。
  - **永久（鑽石）**：每 5 波與死亡時結算鑽石，於「技能地圖」做跨局永久強化（可取消退款的大型節點、不可取消的詛咒節點）。
- 平台：純前端（React + HTML Canvas），手機網頁優先設計，可離線、可部署到 GitHub Pages。
- 存檔：無伺服器。以可複製的「進度代碼」保存（含校驗碼）。

---

## 1.5 遊戲模式（`src/data/modes.js`）

開始遊戲時可選模式（皆可搭配三種難度）：

- **經典 classic**：從第 1 波開始，挑戰最高波次。
- **續戰 headstart**：從「最佳波次 − 10」開始（`HEADSTART_OFFSET=10`），並一次結算「第 1 波到起始波次前一波」的累積金幣（`cumulativeWaveGold`），讓你開場就能做基礎升級、直接衝高紀錄。最佳波次需 > 11 才解鎖。
- **無限生存 survival**：以你最佳波次的敵人強度持續猛攻 5 分鐘（`SURVIVAL_SECONDS=300`），spawn 隨時間越來越密集、每 20 秒一隻首領，比拼這段時間的擊殺數。結束以擊殺數結算鑽石；最佳擊殺另存於本機（不放進可攜進度代碼）。

## 1.6 存檔機制

- **本機自動存檔**：進度（鑽石、技能地圖、最佳波次）以 `localStorage`（key `thetower_save_v3`）自動保存，同裝置/瀏覽器關閉重開都在。最佳生存擊殺另存 `thetower_bestkills`。
- **進度代碼**：可攜的字串，用於換裝置/備份（見第 9 節）。GitHub Pages 只託管程式，不保存個人進度。

## 1.7 音訊與設定（`src/engine/audio.js`、`components/Settings.jsx`）

- 以 Web Audio API 即時合成 **8-bit 音效**（擊殺、首領擊殺、受擊、升級、購買、技能、過波、首領出場、結束）與 **循環背景音樂**（旋律＋低音）。零外部檔案、可離線。
- 行動裝置限制：音訊在使用者**第一次點擊**後才啟動（`audio.resume()`）。
- **設定區**可分開開關「背景音樂」與「音效」，設定持久化於 localStorage。
- 引擎不直接相依音訊：戰鬥事件推入 `g.sounds`，由 `App` 迴圈統一播放（每幀限量）。

## 1.8 技能地圖重置（付費重練）

- 技能地圖可一鍵重置：**清空所有節點（含詛咒）**、退還已花費鑽石，並收取**手續費（花費的 15%，至少 20💎）**。
- 這是繞過「詛咒永久不可取消」的付費手段；`spentDiamonds` / `resetFee` 在 `data/skillTree.js`。

## 2. 操作 / UI

- **主選單**：開始遊戲、技能地圖、數值面板、敵人圖鑑、進度代碼。
- **遊戲內 HUD**：返回、波次、金幣、鑽石、難度、數值面板(📊)、敵人圖鑑(👾)、暫停。下方為塔的生命條。
- **武器列**：切換已解鎖武器（標準彈/追蹤彈/雷射/折射激光/火焰）。
- **主動技能列**：過載、新星、冰霜、修復（各有 CD）。
- **局內升級列**：攻擊/防禦/特殊三分頁，點圖示用金幣升級，長按看詳細數值。
- **戰場**：雙指縮放。
- **技能地圖**：拖曳平移、雙指縮放、點節點看路徑與說明、發光虛線圈代表現在可點亮。

## 3. 核心戰鬥機制

### 3.1 傷害公式（玩家與敵人雙向通用）

```
實際傷害 = max(1, 攻擊力 − 對方防禦力)
暴擊時：攻擊力先 × 1.5，再扣防禦。
```

- 玩家攻擊敵人：`max(1, 塔傷害×(暴擊?1.5:1) − 敵人防禦)`。
- 敵人攻擊玩家：`max(1, 敵人攻擊 − 玩家護甲) × 承受傷害倍率`。
- **離散傷害**（子彈、折射、新星）：每次命中各自套用上式（下限 1）。
- **持續傷害**（雷射、火焰、無人機，以「每秒」計）：`max(0, 每秒傷害 − 防禦) × dt`；
  防禦可把弱攻擊完全擋下（下限 0），這正是「重甲」需要高單發武器（雷射）的原因。
- **荊棘灼燒**：視為環境真實傷害，無視防禦。
- **護盾（護盾兵）**：傷害先扣護盾，溢出才扣血；護盾會緩慢回復。

實作位置：`src/engine/game.js`（`mitigate` / `mitigateDot` / `damageEnemy`）、`src/engine/update.js`（各武器呼叫）。

### 3.2 波次與成長

- 每波敵人數量：`min(22, floor(3 + 波次 × 0.6))`（微幅成長且有上限）。
- 每往後一波，敵人屬性指數成長（`src/data/tuning.js` 的 `CFG`）：
  - 血量 ×1.16/波、攻擊 ×1.07/波、防禦 ×1.04/波、移速 ×1.012/波（移速倍率上限 1.9）。
  - 再乘上難度倍率（簡單/普通/困難）。
- **倍速**：遊戲中可切 1×/2×/4×（以多次模擬子步驟達成，物理穩定）。
- 每 5 波最後一隻為**首領**。
- 過波獎勵金幣：`(10 + 波次×5) × 難度金幣倍率 × 金幣加成`。
- 鑽石：每 5 波 `floor(4 × 難度鑽石倍率 × 鑽石加成)`；死亡時 `floor(波次×2 × …)`。

### 3.3 戰場尺寸（v4 調整）

- 生成半徑放大到 `WORLD.spawnR = 4.0`（舊版 2.8），敵人從更遠處湧入、往中心移動，
  並調整 `viewDiv` 讓整個生成環在預設視野內可見，營造開闊戰場。
- 敵人半徑與移速同步上調以維持可視性與節奏。

## 7. 局內升級（金幣，`TREE`）

**多武器系統（重要）**：所有「已解鎖」的武器會**同時開火**（不再切換單一武器）。每把武器各自獨立加點，全域升級則對塔與所有武器生效。

- **全域(塔)升級**：生命、恢復、護甲、範圍、暴擊。各自單獨計價（`globalItemCost`）。
- **各武器升級（項目因武器而異）**：
  - 標準彈 cannon：傷害、攻速、多重、穿透、濺射、彈速
  - 追蹤彈 homing：傷害、攻速、多重、濺射、彈速
  - 雷射 laser：傷害、多重(光束數)　← 無穿透/攻速（持續性）
  - 折射 chain：傷害、攻速、彈射、彈速
  - 火焰 flame：傷害、火域(範圍)
- **跨武器漲價耦合**：武器升級的價格依「該項目在所有武器的等級總和」指數成長（`weaponItemCost` 用 `weaponItemTotal`）。在 A 武器升某項，B 武器的同項目也會變貴 → 逼你做取捨。
- 子彈基礎速度較低（`WORLD.bulletSpd`），靠各武器「彈速」加回（`bulletSpeed = base × (1+彈速×0.3)`）。
- 資料：`data/skills.js`（`ITEMS`/`GLOBAL_ITEMS`/`WEAPON_ITEMS`/`createSkill`/成本函式）；數值換算見 `engine/stats.js` 的 `derive`（回傳全域數值 + `out.weapons[wk]`）；開火見 `engine/update.js`（`stepGame` 對每把武器各自開火）。

## 8. 永久進度（鑽石，`src/data/skillTree.js` + `relics.js`）

分為四個分頁（`components/PermScreen.jsx`）：

1. **基礎屬性樹 `BASE_TREE`**：全域節點圖，影響塔與所有武器（攻擊力/生命/防禦/暴擊/恢復/金幣/鑽石/視野）。
2. **武器箱 `ARMORY`**：購買/解鎖武器（`weaponsOwned`），並升級各武器「基礎數值」（`weaponBase[wk]`，如基礎傷害/攻速，線性漲價）。
3. **武器樹 `WEAPON_TREE[wk]`**：每把武器一棵專屬樹，**只強化該武器**（含重新設計的專屬特殊節點 `special`）。
4. **道具 `RELICS`**：鑽石永久購買，**一場戰鬥只能裝備一個**（`relicEquipped`），效果為全域加成（力量/生命/壁壘/財富/稜鏡/狂怒/荊棘/軌道/不死）。

- 每把武器有**根本性差異**（`data/weapons.js`）：`dmgF/rateF/rangeF/spd` 各不同、`special` 標示核心特性（穿透/追蹤爆破/持續高 DPS/彈射/近身範圍）。射程逐武器計算（`out.weapons[wk].range`）。
- 數值衍生 `derive(meta, skill)`：全域加成 = 基礎樹 + 裝備道具；每把武器 = 全域 × 該武器樹 × 武器箱基礎 × 局內升級。
- 通用樹渲染 `components/TreeView.jsx`（基礎樹與各武器樹共用）；每棵樹可付費重置（退還 85%）。
- UI：點節點只選取、按鈕才購買（防誤觸）；連線 `non-scaling-stroke` 任何縮放都清楚；節點下方預覽鑽石花費。

## 8.5 局內升級的同類連動漲價

- 全武器同時開火；每把武器各自加點（`data/skills.js` 的 `WEAPON_ITEMS`，因武器特性而不同：雷射無穿透、火焰只有傷害/火域…）。
- **攻擊類連動**：每買任一武器升級一級，全部武器升級價格 ×1.01（`CLASS_MULT`，`attackTotal` 為計次）；全域升級亦有自己的連動。讓金幣分配成為策略。

## 9. 存檔 / 進度代碼（`src/engine/save.js`）

- 本機自動存檔：整份 meta 以 JSON 存於 `localStorage`（前向相容，新增節點不會清空）。
- 可攜進度代碼：整份 meta → JSON → base64，前綴 `T4-` + 校驗碼（`encodeSave`/`decodeSave`）。
- `legacyDecodeV3`：相容舊版位元格式，啟動時遷移鑽石/最佳波次。
- 節點數量變動時提高 `VERSION`，避免讀到不相容的舊格式（目前為 `3`）。

## 10. 程式架構（模組化，各司其職）

```
index.html                 頁面殼層
src/
  main.jsx                 入口：掛載 React
  App.jsx                  協調者：持有 ref/狀態、跑遊戲迴圈、切畫面與彈窗
  styles.js                共用樣式常數

  data/                    純資料（無邏輯，易平衡）
    difficulty.js          難度
    tuning.js              成長曲線 CFG、世界尺寸 WORLD、縮放範圍
    enemies.js             敵人定義 + pickType
    weapons.js             武器定義
    skills.js              局內升級 TREE + 主動技能 ABILITIES
    skillTree.js           永久技能地圖 NODES + 輔助函式（含 childrenOf 預覽）

  engine/                  機制（不碰 React/畫布，可單獨測試）
    stats.js               derive 數值衍生、CRIT_MULT、enemyStatsAt
    save.js                進度代碼編碼/解碼
    game.js                建立遊戲、生成、擊殺、傷害公式、特效資料
    update.js              每幀模擬 stepGame、rangeOf、主動技能 triggerAbility

  render/                  動畫/渲染
    shapes.js              敵人形狀路徑
    draw.js                每幀畫布繪製（漸層、外發光、AOE 光環）

  components/              畫面（React 元件）
    Icon.jsx Menu.jsx Overlay.jsx Hud.jsx GameScreen.jsx
    WeaponBar.jsx AbilityBar.jsx SkillBar.jsx
    SkillMap.jsx StatsPanel.jsx EnemyPanel.jsx CodesOverlay.jsx
    widgets.jsx
```

資料流：`App` 用 `ref` 持有可變遊戲狀態 `g` 與衍生數值 `s`；
迴圈每幀呼叫 `stepGame(g, s, dt, weapon, io)` 推進模擬、`draw(...)` 繪製，
並節流地把摘要同步到 React state 餵給 HUD。跨局鑽石/最佳波次透過 `io` 回寫 `metaRef`。

## 11. 玩家（產品擁有者）的期望與路線圖

> 這節記錄需求方對遊戲的願景，供後續開發對齊方向。

- **畫面更精美**：不要只靠粒子，要用漸層、光暈、發光、AOE 光環等讓畫面更美（持續精進中）。
- **超大且完整的遊戲**：內容豐富、技能多、可長期養成。
- **手機網頁優先**：觸控操作、可雙指縮放、版面適配手機。
- **明確的數值體驗**：玩家要能清楚看到「我點了什麼 → 數值變成怎樣」，並能比較「卡在第 N 波，需要把傷害堆到多少」。
- **能在 GitHub 上直接玩**：透過 GitHub Pages 部署。
- **架構乾淨**：模組化、各司其職、輕量、易維護；要有說明區（本檔）與變更紀錄區（`changelog/`）。

### 待辦 / 可持續強化的方向

- 更多敵人類型與機制（遠程、治療、召喚、菁英詞綴）。
- 更多武器與主動技能、武器專屬升級。
- 畫面再升級：受擊閃光、螢幕震動、擊殺特效、背景星空/視差。
- 音效與音樂。
- 成就 / 統計 / 每日種子挑戰。
- 數值平衡持續調整（集中在 `data/` 方便調參）。

## 12. 開發 / 部署

```bash
npm install
npm run dev       # 本機開發
npm run build     # 產出 dist/
npm run preview   # 預覽 build
```

GitHub Pages：`.github/workflows/deploy.yml` 會在 push 到 `main` 時自動 build 並發佈
（`configure-pages` 設 `enablement:true`，首次會自動開啟 Pages）。
網址格式：`https://<帳號>.github.io/<repo>/`。

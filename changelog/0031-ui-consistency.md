# 0031 · UI 一致性與清晰度（設計系統統一）

- 日期：2026-05-29

## 摘要
建立共用設計系統並套用全畫面，讓分頁、按鈕、卡片、用語一致明確。

## 細項
- `styles.js`：新增統一色票 `C`、卡片 `card`、**分頁 `tabStyle(active, accent, fixedWidth)`**、**按鈕 `btn(variant)`**（primary/secondary/danger/ghost）。
- 全畫面分頁統一外觀（同樣的圓角/邊框/選中態）：局內升級的武器分頁與攻/防/特、永久進度四分頁與武器子分頁、敵人圖鑑難度分頁。
- 暫停選單與結算按鈕改用統一 `btn()`（主要動作 primary、其餘 secondary）。
- 用語統一：主選單「技能地圖」→「永久進度」（與面板標題一致）。

## 測試
- `npm run build` 通過；headless 模擬器仍正常運行（引擎未受影響）。

## 影響範圍
- 純 UI/樣式；不影響數值或存檔。

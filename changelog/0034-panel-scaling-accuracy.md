# 0034 · 敵人面板數值準確性（對齊實戰成長曲線）

- 日期：2026-05-30

## 摘要
更深一層稽核「面板顯示 vs 實際機制」，修掉兩個會誤導玩家的數值不一致。

## 修正
1. **面板謊報成長曲線**：敵人圖鑑底部硬寫「血量 ×1.16、攻擊 ×1.07、防禦 ×1.04、
   移速 ×1.015」，但 `tuning.js` 的曲線早已調成 **×1.12 / ×1.07 / ×1.022 / ×1.012**
   （放緩後讓永久投入更有回報）。改為**直接讀 `CFG` 動態顯示**，並補上「移速最高 ×1.9」，
   日後再調參數也不會再對不上。
2. **面板移速未套上限**：`enemyStatsAt()`（敵人面板計算用）算移速時沒有套用
   `spdScaleCap`，但實戰移動有套。導致後期（約 55 波後）面板會**高估**敵人移速。
   已對齊實戰：`min(spdScaleCap, base^(wave-1))`——驗證第 99 波步兵 = 0.185×1.9 ≈ 0.3515，
   與第 60 波相同（封頂不再成長）。

## 測試
- `npm run build` 通過（68 模組）。
- 數值驗證：步兵移速 w1=0.185 / w60=0.296 / w99=0.3515（封頂），面板與實戰一致。
- headless 模擬器曲線穩定（首玩約 28、全投入 41–54 波）。

## 註記（誠實記錄）
本回合稽核過程中，我一度誤判 `render/draw.js` 為死碼。經實際檢查，`draw.js`
是現役渲染器（`App.jsx` 直接 import、每幀呼叫），**並未刪除**，其 save/restore
亦成對平衡，無問題。沒有檔案被誤刪。

## 影響範圍
- `engine/stats.js`、`components/EnemyPanel.jsx`：僅面板顯示準確性；不改存檔、不改平衡。

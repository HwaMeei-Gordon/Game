# 0033 · 深度嚴格稽核 + 修正持續型武器暴擊（雷射／火焰）

- 日期：2026-05-30

## 摘要
針對整套技能／升級／戰鬥邏輯做更深、更嚴格的交叉稽核，找出一個
「看似有用、實際上幾乎無效」的設計：雷射／火焰的暴擊投資，並修正它。

## 深度稽核結果（逐項驗證）
- **節點完整性**：全部 7 棵樹 0 重複 id、0 壞 parent、0 無效 key；`App.jsx` 的
  `NODE_INDEX` 由 BASE_TREE + 全武器樹建立，**每個新節點都可購買**。
- **圖示**：技能用到的 icon 全部存在於 `Icon.jsx`（且有 `dmg` 後備），不會出現空白圖示。
- **道具**：`relics.js` 全部 bonus／flag（dmgM/hpM/armor/goldM/gem/critC/thorns/orbs/
  lifesteal/takeDmg、glass/fortress/immortal）都被 `stats.js`／`update.js` 消耗，無死碼。
- **穿甲流向**：armorPen 正確套用到雷射、火焰、折射、碎片、濺射 AOE。
- **AOE 擊殺**：濺射／爆破打死的鄰近敵人會延遲一幀由移動迴圈的 `hp<=0` 收掉，
  仍正常結算金幣與分裂，沒有殘留。

## 修正：雷射／火焰暴擊形同虛設 → 改為期望值穩定加成
- 問題：兩者為「持續結算」武器，原本暴擊用 `Math.random() < 暴擊率 × 0.3` 判定，
  等於把暴擊率打三折，每個暴擊節點／道具實際只貢獻約 1% DPS——投資幾乎沒意義。
- 修正：持續型武器改用**期望值**穩定加成 `傷害 ×(1 + 暴擊率 ×0.5)`，每跳穩定生效。
  這讓「基礎樹暴擊／武器樹暴擊節點／狂怒核心」對雷射火焰**真正有效且可感知**，
  且因為用的是隨機暴擊的數學期望值，總體強度等價、不會暴衝。

## 測試
- `npm run build` 通過；節點驗證腳本全綠。
- headless 模擬器：首玩約 28 波、全永久投入 41–55 波，與修正前一致（此修正為
  價值修復而非強度灌水，故總曲線穩定）。

## 影響範圍
- `engine/update.js`（雷射／火焰傷害結算）。不改存檔，不改數值面板公式。
